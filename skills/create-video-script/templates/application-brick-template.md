## BES Framework

Every step must be **Big**, **Easy**, and **Specific**.

| Quality | Question to Ask | Pass/Fail |
|---------|-----------------|-----------|
| **Big** | Will this make a meaningful difference? | |
| **Easy** | Can viewers confidently implement this? | |
| **Specific** | Could anyone follow this without confusion? | |

- [ ] Not too simple for experts / not too complex for beginners

## Application Brick Checklist

- [ ] Smooth transition from Example Brick
- [ ] All steps pass BES test
- [ ] Steps broken down small enough
- [ ] Matched to audience level
- [ ] Show Don't Tell continues where possible
- [ ] No vague advice ("just learn Python")
- [ ] No overly complex instructions
- [ ] Transition out creates curiosity (no summary)

**Ultimate Test:**
> "If viewers implemented what I've shown them, would they actually get a result?"

[ ] Yes → Brick is complete
[ ] No → Revise steps to be more actionable
